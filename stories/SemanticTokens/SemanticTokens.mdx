## Usage Guidelines

Semantic tokens provide meaning and context to your colors:

- **Use semantic tokens instead of base colors** in component styles
- **Base colors define the palette**, semantic tokens define the meaning
- **Semantic tokens can be remapped** for themes or brand variations
- **They create consistency** across different UI contexts

### Example Usage
```css
.button-primary {
  background-color: var(--md-color-cta-primary-default);
  color: var(--md-color-text-inverse);
}

.button-primary:hover {
  background-color: var(--md-color-cta-primary-hover);
}

.card {
  background-color: var(--md-color-surface-secondary);
  border: 1px solid var(--md-color-border-secondary);
  color: var(--md-color-text-primary);
}
```
